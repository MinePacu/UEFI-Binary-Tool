# GUI 사용법

## 실행

프로젝트 루트에서 다음 명령을 실행합니다.

```bash
python3 gui_main.py
```

또는 패키지 모듈로 실행합니다.

```bash
python3 -m uefi_binary_tool
```

Python에 Tkinter가 포함되어 있으면 데스크톱 GUI가 열립니다. Tkinter가 없으면 로컬 웹 UI가 실행되며, 터미널에 표시된 `http://127.0.0.1:포트/` 주소가 브라우저로 열립니다.

## 언어 선택

UI 언어는 기본적으로 운영체제 언어 설정을 따라갑니다.

- 한국어 OS: 한국어 UI
- 그 외 언어: 영어 UI

언어를 강제로 지정하려면 환경 변수를 사용합니다.

```bash
UEFI_BINARY_TOOL_LANG=ko python3 gui_main.py
UEFI_BINARY_TOOL_LANG=en python3 gui_main.py
```

## 화면 구성

GUI는 다음 영역으로 구성됩니다.

- 제조사 선택: `ASUS` 또는 `MSI`
- 분석 탭: BIOS/Section 파일을 분석
- 리패킹 탭: 원본 파일, 추출 이미지 디렉터리, 출력 파일 경로를 지정해 리패킹
- 작업 로그: 검증, 분석, 리패킹 진행 상황과 출력 파일 경로 표시
- 상태 표시줄: 현재 작업 상태 표시

[GUI 메인 창에서 제조사 콤보박스, 분석 탭, 리패킹 탭, 작업 로그가 함께 보이는 장면]

## 분석 실행

1. 제조사를 선택합니다.
2. `분석` 탭을 엽니다.
3. `BIOS/Section 파일`에 분석할 파일을 지정합니다.
4. `분석 시작`을 누릅니다.
5. 작업 로그와 완료 메시지를 확인합니다.

[ASUS를 선택하고 분석 탭에서 BIOS/Section 파일 경로를 입력한 뒤 분석 시작 버튼을 누르기 직전의 장면]

분석 성공 시 생성되는 파일:

| 제조사 | 생성 파일 |
|---|---|
| ASUS | `<원본파일명>_analysis.txt`, `<원본파일명>_analysis.md` |
| MSI | `<원본파일명>_msi_analysis_report.txt` |

## 리패킹 실행

1. 제조사를 선택합니다.
2. `리패킹` 탭을 엽니다.
3. `원본 BIOS 파일`을 지정합니다.
4. `추출 이미지 디렉터리`를 지정합니다.
5. `출력 파일`을 지정합니다. 비워두면 기본 이름이 자동 생성됩니다.
6. `리패킹 시작`을 누릅니다.
7. 작업 로그의 `[OUTPUTS]` 또는 `[OUTPUT]` 영역에서 생성 파일을 확인합니다.

[리패킹 탭에서 원본 BIOS 파일, 추출 이미지 디렉터리, 출력 파일 경로가 모두 입력된 장면]

기본 출력 파일 이름:

| 제조사 | 기본 출력 이름 |
|---|---|
| ASUS | `<원본파일명>_asus_repacked.bin` |
| MSI | `<입력디렉터리명>_msi_repacked.bin` |

MSI 리패킹은 원본 BIOS 파일을 비워둘 수도 있습니다. 다만 원본 파일을 지정하면 원본 분석 결과를 사용해 구조 보존과 변경 감지가 더 정확해집니다.

## 웹 UI 사용 시 차이점

웹 UI는 브라우저 보안 제한 때문에 파일 선택 창으로 로컬 경로를 프로그램에 전달하지 않습니다. 각 입력 칸에 전체 경로를 직접 입력하세요.

[브라우저에서 열린 웹 UI에 파일 경로를 직접 입력하는 장면]

웹 UI는 다음 API를 내부적으로 사용합니다.

- `POST /api/run`: 분석 또는 리패킹 시작
- `GET /api/status`: 상태와 로그 갱신

일반 사용자는 API를 직접 호출할 필요가 없습니다.
